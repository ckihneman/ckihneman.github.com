# Debugging 101
## Don't hurt yourself with $( '.some-selector' ).on( 'click', function() { alert( 'the pain.' ); });
## Help yourself with $( 'div' ).on( 'click', function( e ) { console.log( 'something useful.' ); console.log( 'two aruments or more', this, e, e.type ); // view the element as its html console.log( this ); // inspect the element as an object, very useful // for digging into the internals of JavaScript console.dir( this ); });
## **`$0`** is your friend So awesome - 1. Inspect an element 2. Hit escape to activate your console as well 3. Type **`$0`** into the console. To get a jQuery object, just do **`$($0)`**. Boom! Now you can test something on the fly if you please. If you are in `jQuery.noConflict()` (as some CMS's do by default, Wordpress), then **jQuery($0)**.
## OMG Nothing is happening * Did you get your selector right? * Check `$('.some-selector').length` (maybe in a `console.log` call) to ensure you didnt either mess up your selector, or not put the element in your html. * Do you have other plugins on the page? * Sometimes plugins will prevent events from bubbling and could be blocking what you are trying to accomplish. Sometimes, you just have to read through a plugins code...

# Don't use `.live`... Just be more specific... $( document ).on( 'click', 'a', function( e ) { console.log( $(this).attr( 'href' ) ); }); But only do that if you really have to...it's going to always be better to delegate to the closest element possible.
## Getting `this` What is `this`? $( '.box' ).on( 'click', function() { // `this` vs $( this ) var $this = $( this ); console.log( 'this', this ); console.log( '$( this )', $this ); console.dir( this ); console.dir( $this ); }); In event callbacks, `this` is the native dom element being acted on. That's why if you want to do jQuery operations you have to do `$( this )`, then you can [have your way with the element](blank.html).
# Common pitfalls
# Don't call jQuery unless you have to. Bad $( '.box' ).text( 'awesome box' ); $( '.box' ).css( 'color', 'blue' ); $( '.box' ).addClass( 'visible' ); Good var $box = $( 'box' ); $box.text( 'awesome box' ); $box.css( 'color', 'blue' ); $box.addClass( 'visible' );
# Even better... var $box = $( 'box' ); $box .text( 'awesome box' ) .css( 'color', 'blue' ) .addClass( 'visible' ); Remember, jQuery methods always (almost always) return themselves...chain it!
### Don't make sizzle work harder than it needs to! Sizzle is the dom query engine in jQuery, it's great and all, but dont overwork it if you don't have to. Instead of just `$( '.box' )` ...which searches your entire page... Be more specific... $( '#content' ).find( '.box' ); // or $( '#content .box' ); // or $( '.box', $('#content') ); But just use `$( '#content' ).find( '.box' );`. Its readable, and a lot of the time you might need to cache `#content` anyways...
